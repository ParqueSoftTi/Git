Para instalar Git en tu sistema, sigue estos pasos:

  1. Visita el sitio web oficial de Git: git-scm.com.
  2. Descarga la versión adecuada para tu sistema operativo (Windows, macOS, Linux).
  3. Sigue las instrucciones de instalación proporcionadas en el sitio web.
  4. Una vez instalado, configura tu nombre de usuario y dirección de correo electrónico utilizando los siguientes comandos: 
    git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

En esta práctica, aprenderás los conceptos básicos de Git trabajando en un repositorio local. Sigue estos pasos:

  1. Crea un nuevo repositorio Git en tu sistema local utilizando el comando git init.Para iniciar un nuevo repositorio Git, simplemente navega hasta la carpeta raíz de tu proyecto en tu terminal o línea de comandos y ejecuta el siguiente comando:
   git init

Esto creará un nuevo repositorio Git en esa carpeta.

  1. Añade archivos al repositorio utilizando git add y realiza tu primer commit con git commit.Después de inicializar el repositorio, puedes comenzar a agregar archivos al área de preparación (staging area) utilizando el comando git add. Por ejemplo, para agregar todos los archivos en el directorio actual, puedes usar:
   git add .

Luego, realiza tu primer commit con un mensaje descriptivo utilizando el comando git commit:

   git commit -m "Mensaje descriptivo del commit"

Esto guardará los cambios en el repositorio Git.

  1. Crea y trabaja en ramas nuevas utilizando git branch y git checkout.Puedes crear una nueva rama en Git utilizando el comando git branch seguido del nombre de la nueva rama. Por ejemplo, para crear una nueva rama llamada "nueva-caracteristica", puedes usar:
   git branch nueva-caracteristica

Luego, cambia a la nueva rama utilizando el comando git checkout:

   git checkout nueva-caracteristica

Ahora puedes realizar cambios en los archivos y hacer commits en esta nueva rama sin afectar la rama principal.

  1. Realiza cambios en los archivos, haz commits en la rama actual y fusiona ramas utilizando git merge.Después de realizar cambios en los archivos y hacer commits en la rama actual, puedes fusionar los cambios de una rama a otra utilizando el comando git merge. Por ejemplo, para fusionar los cambios de la rama "nueva-caracteristica" a la rama principal, primero cambia a la rama principal:
   git checkout main

Luego, realiza la fusión:

   git merge nueva-caracteristica

Esto integrará los cambios de la rama "nueva-caracteristica" en la rama principal.

En esta práctica, aprenderás a trabajar con repositorios remotos en GitHub. Sigue estos pasos:

  1. Crea un nuevo repositorio en GitHub.
    • Inicia sesión en tu cuenta de GitHub.
    • Haz clic en el botón "New" para crear un nuevo repositorio.
    • Sigue las instrucciones para configurar tu repositorio remoto.
  2. Clona el repositorio remoto a tu sistema local utilizando git clone.Una vez que hayas creado el repositorio en GitHub, puedes clonarlo en tu sistema local utilizando el siguiente comando:
   git clone URL-del-repositorio

Esto creará una copia del repositorio remoto en tu sistema local.

  1. Realiza cambios locales, haz commits y envía tus cambios al repositorio remoto utilizando git push.Después de clonar el repositorio, puedes realizar cambios en los archivos, agregarlos al área de preparación y hacer commits de la misma manera que lo harías en un repositorio local. Una vez que estés listo para enviar tus cambios al repositorio remoto en GitHub, utiliza el comando git push:
   git push origin main

Esto enviará tus cambios a la rama principal del repositorio remoto en GitHub.

  1. Trabaja con ramas en el repositorio remoto, crea solicitudes de extracción (pull requests) y fusiona cambios utilizando la interfaz de GitHub.En GitHub, puedes trabajar con ramas de la misma manera que lo haces localmente. Puedes crear nuevas ramas, realizar cambios, hacer commits y fusionar ramas utilizando la interfaz web de GitHub. Además, puedes crear solicitudes de extracción (pull requests) para proponer cambios en el repositorio remoto y fusionarlos con otras ramas. Utiliza la interfaz de GitHub para revisar los cambios, discutirlos con tu equipo y fusionarlos cuando estés listo.

Paso 1: Preparación del Repositorio Clona el repositorio principal.

git clone <url_del_repositorio>
cd <nombre_del_repositorio>

Crea una nueva rama para trabajar.

git checkout -b feature-nueva-funcionalidad

Realiza tus cambios en la nueva rama y haz commit de ellos.

git add .
git commit -m "Implementa nueva funcionalidad"

Paso 2: Fusión Utilizando Squash La fusión con squash combina todos los commits de la rama de características en un solo commit antes de fusionarlos con la rama principal. Esto ayuda a mantener un historial de confirmaciones más limpio y coherente.

git checkout main
git merge --squash feature-nueva-funcionalidad
git commit -m "Implementa nueva funcionalidad"

Paso 3: Fusión Utilizando Rebase Rebasing reorganiza el historial de confirmaciones moviendo los commits de una rama a otra. Esto ayuda a mantener un historial lineal y limpio, pero ten en cuenta que puede reescribir el historial de confirmaciones, por lo que se debe usar con cuidado.

git checkout main
git rebase main feature-nueva-funcionalidad
git checkout main
git merge feature-nueva-funcionalidad

Paso 4: Fusión Utilizando Merge El merge es la técnica más simple y común para fusionar ramas en Git. Simplemente combina los cambios de una rama en otra y crea un nuevo commit de fusión.

git checkout main
git merge feature-nueva-funcionalidad

Paso 5: Eliminación de la Rama de Características Después de fusionar la rama de características con éxito, es una buena práctica eliminarla para mantener el repositorio limpio.

git branch -d feature-nueva-funcionalidad
